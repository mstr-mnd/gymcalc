# Деплой на сервер (serv.host) с доменом nic.ru

Сайт упакован в Docker. Запускается одной командой. HTTPS-сертификат
получается автоматически (через Caddy), сам обновляется — вручную ничего настраивать не нужно.

---

## Что понадобится
- VPS на serv.host с Ubuntu 22.04+ и его IP-адрес (например `123.45.67.89`).
- Доступ по SSH (логин/пароль или ключ от serv.host).
- Домен `gymscalp.online` в панели nic.ru.
- GA4 ID: `G-TDSDP350RW`.

---

## Шаг 1. Направить домен на сервер (панель nic.ru)
Открой управление DNS для `gymscalp.online` и создай запись:

| Тип | Имя (хост) | Значение         |
|-----|------------|------------------|
| A   | `@`        | IP твоего сервера |

(Если хочешь ещё `www` — добавь вторую A-запись с именем `www` и тем же IP,
и раскомментируй блок `www` в файле `Caddyfile`.)

DNS обновляется не мгновенно. Через 10–60 минут проверь с любого компьютера:
```bash
ping gymscalp.online
```
Должен отвечать IP твоего сервера. **Не переходи к запуску, пока это не так** —
иначе Caddy не сможет выпустить сертификат.

---

## Шаг 2. Подключиться к серверу
С твоего компьютера:
```bash
ssh root@IP_СЕРВЕРА
```
(IP и пароль — из письма serv.host.)

---

## Шаг 3. Установить Docker
На сервере:
```bash
curl -fsSL https://get.docker.com | sh
```
Проверка:
```bash
docker --version
docker compose version
```

---

## Шаг 4. Загрузить проект на сервер
**Вариант А — через Git** (удобно обновлять потом). Если выложишь проект на GitHub:
```bash
git clone https://github.com/ТВОЙ_АККАУНТ/ТВОЙ_РЕПО.git gymscalp
cd gymscalp
```

**Вариант Б — скопировать с компьютера.** На своём компьютере (не на сервере),
из папки выше проекта:
```bash
rsync -av --exclude node_modules --exclude .next \
  /Users/artyom/Projects/prjct/ root@IP_СЕРВЕРА:/root/gymscalp/
```
Затем на сервере: `cd /root/gymscalp`.

---

## Шаг 5. Создать файл `.env`
В папке проекта на сервере:
```bash
echo "NEXT_PUBLIC_GA_ID=G-TDSDP350RW" > .env
```
Этот ID «вшивается» в сборку, поэтому он должен быть тут **до** запуска.

---

## Шаг 6. Открыть порты
```bash
ufw allow 22
ufw allow 80
ufw allow 443
ufw --force enable
```

---

## Шаг 7. Запуск
```bash
docker compose up -d --build
```
Первый раз: соберётся образ (2–4 минуты) и Caddy получит SSL-сертификат.

---

## Шаг 8. Проверить
1. Открой `https://gymscalp.online` — сайт работает, замок HTTPS на месте.
2. Нажми «Принять» в баннере cookie.
3. В Google Analytics → **Reports → Realtime** появится твой визит.

Если страница не открывается сразу — подожди минуту (Caddy выпускает сертификат)
и обнови. Посмотреть, что происходит:
```bash
docker compose logs -f
```

---

## Обновление сайта после изменений в коде
**Если деплоил через Git:**
```bash
git pull
docker compose up -d --build
```
**Если копировал через rsync** — повтори команду `rsync` из шага 4, затем:
```bash
docker compose up -d --build
```

> Важно: если меняешь GA-ID или любой код — нужен именно `--build`,
> просто `restart` не пересоберёт сайт.

## Полезные команды
| Действие                | Команда                          |
|-------------------------|----------------------------------|
| Логи (живые)            | `docker compose logs -f`         |
| Перезапустить           | `docker compose restart`         |
| Остановить              | `docker compose down`            |
| Пересобрать и поднять   | `docker compose up -d --build`   |
| Статус контейнеров      | `docker compose ps`              |

---

## Частые проблемы
- **Сайт не открывается / нет сертификата.** Скорее всего DNS ещё не указывает на сервер,
  или закрыты порты 80/443. Проверь `ping gymscalp.online` и шаг 6.
- **Аналитики нет.** Проверь, что файл `.env` создан (шаг 5) и ты пересобрал
  с `--build`. И не забудь нажать «Принять» в баннере cookie — без согласия GA не грузится.
- **Поменял домен.** Поправь домен в `Caddyfile` и в `lib/i18n.ts` (`SITE_URL`),
  затем `docker compose up -d --build`.
